# 项目
请参考package.json确定技术栈。

# 规则
代码准则：极致性能、高复用性、高可读性、易维护，逻辑嵌套≤3 层，分区分块编写
语法规范：优先使用let、禁用var；不可变对象用const；函数不使用const声明
规范：所有入参、返回值、字段、函数，必须通过/** */添加 JSDoc 注释
格式：合理换行，代码精简无冗余

# 流程步骤
1.拉取 `项目模板`:`https://github.com/QLing-yes/ElysiaTemplate.git`
2.根据用户选择的选项，从`./template/`的根目录匹配对应的模板目录，将内容复制到`项目模板`根目录下并覆盖已存在的文件。
3.根据用户选择的选项，从`./operation.js`导入对应的操作函数并执行。
